Kiến Trúc Kỹ Thuật Tiên Tiến

SuperSearch hoạt động theo mô hình Serverless Offline Desktop App kết hợp sức mạnh xử lý ngôn ngữ cục bộ của Python và hiệu năng hiển thị tối ưu của thư viện WebEngine.

1. Pipeline Chuyển Đổi Tài Liệu (Microsoft MarkItDown)

Ứng dụng tích hợp các bộ chuyển đổi tùy chỉnh kế thừa từ DocumentConverter nhằm chuẩn hóa mọi định dạng tệp về Markdown trước khi lập chỉ mục:

  • LocalOcrPdfConverter: Trích xuất văn bản gốc bằng pdfplumber. Nếu phát hiện tệp PDF là dạng quét (chứa ít hơn 100 ký tự), hệ thống tự động gọi engine Tesseract-OCR cục bộ được đóng gói sẵn để quét và nhận diện chữ Việt/Anh (vie+eng).
  • LocalOcrImageConverter: Tự động nhận diện chữ quang học trên các tệp ảnh (PNG, JPG, JPEG) bằng công nghệ OCR đa luồng.
  • LocalDocConverter (Heuristic Anti-CJK): Đọc tệp Word cũ (.doc) bằng olefile, giải mã UTF-16LE, khôi phục các ký tự ANSI lỗi và lọc bỏ hoàn toàn các ký tự nhiễu tiếng Trung CJK để trích xuất văn bản tiếng Việt/Anh chính xác.
  • LocalXlsConverter: Đọc bảng dữ liệu Excel cũ (.xls) bằng xlrd và chuyển đổi cấu trúc thành các bảng Markdown tiêu chuẩn.

2. Quản Lý Tài Nguyên Thông Minh (RAM-Aware Multi-threading)

Nhằm tránh gây treo hoặc đơ hệ thống trong quá trình đánh chỉ mục dữ liệu lớn, SuperSearch áp dụng cơ chế tự động giới hạn luồng động:

  • Giám Sát RAM API: Sử dụng Windows Kernel API (GlobalMemoryStatusEx) qua thư viện ctypes để liên tục đo đạc tải lượng RAM hệ thống.
  • Tự Điều Chỉnh Worker: Nếu RAM của máy tính vượt quá 70% tải, luồng xử lý tự động hạ về chế độ đơn luồng (1 worker). Ngược lại, khi máy tính rảnh rỗi, hệ thống sẽ sử dụng tối đa 70% tổng số nhân CPU (ThreadPoolExecutor) để tăng tốc xử lý song song.
  • Điều Khiển Luồng: Hỗ trợ các cơ chế kiểm soát tiến trình thời gian thực bao gồm Tạm dừng (Pause), Tiếp tục (Resume) và Hủy (Abort).

3. Cơ Sở Dữ Liệu Serverless Offline (JavaScript Indexed File)

Thay vì cài đặt các cơ sở dữ liệu cồng kềnh (như Elasticsearch, SQLite), SuperSearch lưu trữ chỉ mục dưới dạng tệp JavaScript tĩnh:

  • Phân vùng chỉ mục: Mã băm MD5 của đường dẫn thư mục quét được sử dụng để đặt tên tệp chỉ mục riêng biệt search_db_[md5_hash].js, tránh ghi đè chỉ mục giữa các thư mục khác nhau.
  • Tải tức thì (0ms Latency): Giao diện Webview có thể nạp trực tiếp chỉ mục này qua thẻ script, cho phép tìm kiếm offline toàn văn (Full-text search) siêu tốc không cần thông qua mạng hay database engine trung gian.

Hướng Dẫn Vận Hành

Hướng dẫn các thao tác quản trị và hệ thống phím tắt nhanh giúp tối ưu hiệu quả sử dụng ứng dụng SuperSearch.

Quy Trình Quét Chỉ Mục Mới

  • Bước 1: Khởi động: Nhấp đúp vào tệp SuperSearch.exe để chạy ứng dụng.
  • Bước 2: Chọn thư mục: Nhấp vào biểu tượng thư mục ở góc trên giao diện, chọn thư mục chứa tài liệu cần quét.
  • Bước 3: Thực hiện quét: Hệ thống tự động chuyển đổi các file tài liệu và tạo tệp chỉ mục. Quá trình quét được cập nhật tiến độ thời gian thực trên giao diện.
