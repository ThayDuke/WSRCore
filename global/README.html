Các lệnh vận hành lõi nâng cấp

Phân luồng brainstorm giúp tối ưu hóa tiến trình làm việc và đảm bảo an toàn mã nguồn.

Lệnh /br (Brainstorm)

Thảo luận logic đơn giản, tối ưu thuật toán cục bộ trực tiếp tại chat.

Sử dụng các tags ẩn: [HIỂU_YÊU_CẦU] (alert TIP), [PHƯƠNG_PHÁP] (alert NOTE), [ĐỀ_XUẤT_TỐI_ƯU] (văn bản thường), [CẢNH_BÁO] (alert WARNING), [NEO_HỒI_QUY] (văn bản thường). Cuối phản hồi bắt buộc hiển thị: VUI LÒNG XEM XÉT VÀ PHÊ DUYỆT!.

Lệnh /pl (Planning)

Chuyên môn hóa cho nhiệm vụ lớn, tái cấu trúc hoặc sửa đổi trên nhiều file.

Tạo file MD tại DOCS/planning_[tên_nhiệm_vụ]_v[X].md chứa YAML Frontmatter, checklist phân loại thay đổi (MODIFY, NEW, DELETE) và mã nguồn đầy đủ chuẩn Senior. Cấm tự ý sửa code khi chưa được duyệt trực tiếp trên chat.

Lệnh /clean (Clean)

Quét dự án và đề xuất danh sách các tệp tin tạm, rác để dọn dẹp tối ưu dung lượng.

Hoạt động ở chế độ đề xuất trước, chờ duyệt từ người dùng. Tuyệt đối không tự ý xóa bất kỳ file nào trước khi được xác nhận trong chat.

Lệnh /commit (Commit)

Hỗ trợ commit các thay đổi đã sửa đổi hoặc toàn bộ dự án vào hệ thống Git.

Cấm Agent tự ý chạy lệnh git commit ngầm. Chỉ thực hiện khi người dùng yêu cầu trực tiếp. Sử dụng Conventional Commit Message siêu ngắn (dưới 10 từ).

Lệnh /reload (Reload)

Nạp lại và ghi nhớ toàn bộ cấu hình, quy tắc, kỹ năng và bài học từ thư mục .codex.

Hỗ trợ 4 cấp độ nạp: Tiêu chuẩn (mặc định), Chỉ định (/reload <skill>), Debug (/reload debug) và Đầy đủ (/reload full) để tối ưu hóa context window.

Lệnh /backup (Backup)

Sao lưu toàn bộ dự án (loại trừ .git) thành file ZIP vào thư mục chỉ định.

Lưu trữ tại C:\Backup\[ProjectName]. Hỗ trợ phím tắt backup [project]* / bak [project]*. Đặt tên tùy chọn hoặc định dạng mặc định [PROJECT]_ddMMyyyy_HHmmss.zip.

Kỹ năng Effective HTML (ehtml / htmle)

Quy chuẩn tạo lập các tài liệu HTML tự chứa, sơ đồ hệ thống SVG và kế hoạch trực quan hóa để kiểm tra nhanh tiến trình vận hành.

3 Sub-skills chính

1. htmls (HTML Self)

Tạo file HTML tự chứa (báo cáo, slide, explainer, prototype) gọn gàng, trực quan và tương tác tốt.

2. htmld (HTML Diagram)

Vẽ sơ đồ kiến trúc hệ thống, stack bằng HTML/SVG chất lượng cao, tối ưu hóa chế độ toàn màn hình.

3. htmlp (HTML Plan)

Tạo trang kế hoạch thực tế, đơn giản, tổ chức trực quan, câu từ ngắn gọn bám sát yêu cầu.

Quy tắc gọi skill trong chat

Khi cần Agent tạo các tài liệu trực quan, người dùng có thể kích hoạt bằng cách gõ: 

"sử dụng skill htmls để tạo báo cáo..."
"sử dụng skill htmld vẽ sơ đồ..."
"sử dụng skill htmlp lập kế hoạch..."

Mô hình sẽ tự động kích hoạt tri thức cấu hình của effective-html (viết tắt ehtml hoặc htmle) để tạo ra output chất lượng cao nhất.

Context Checkpoint & Pruning

Cơ chế tự động lưu checkpoint trạng thái hệ thống, giải phóng bộ nhớ chat cũ và phục hồi nhanh.

MÔ PHỎNG NGỮ CẢNH CHAT ĐẦY (100k tokens)

Nén và làm sạch ngữ cảnh

Sau khi code xong, Agent tự động tổng hợp trạng thái vào file .codex/memory/project_checkpoint.yaml. 

# project_checkpoint.yaml
Last_Task_Completed: "Refactor logic"
Updated_Architecture:
  - File_A: "Update regex logic"
Active_Memory_Anchors:
  - "Local server port: 6868"

Người dùng chỉ cần xóa chat cũ của IDE, mở chat mới và gõ /reload. Agent tự phục hồi nhanh ngữ cảnh mà không cần copy YAML thủ công.

Auto-Memory Consolidation

Khi hoàn thành task, Agent tự động đúc kết lỗi hoặc kinh nghiệm và ghi thêm 1 dòng vào file .codex/memory/AG_LESSONS.jsonl dưới dạng JSONL để ngăn lỗi lặp lại.

Lệnh /reload Phân Cấp & Nạp Động

Cơ chế nạp lại cấu hình chia theo 4 cấp độ để tối ưu hóa bộ nhớ context window của Agent. Hover vào các khối dưới đây để xem chi tiết.

Mặc định

1. Tiêu chuẩn

Chỉ nạp rules cốt lõi và project_checkpoint.yaml. Tiết kiệm token tối đa.

/reload <skill>

2. Chỉ định

Nạp core rules + duy nhất kỹ năng được chỉ định (Ví dụ: /reload stitch).

/reload debug

3. Debug & QA

Nạp core rules + Playbook gỡ lỗi, Checklist hồi quy và Prompt patterns.

/reload full

4. Đầy đủ

Nạp toàn bộ cấu hình, 5 file skills toàn cục và cục bộ, và nhật ký bài học AG_LESSONS.jsonl.

Cơ chế Dynamic Skill Selection trong /br

Khi chạy lệnh /br, Agent tự động nhận diện phạm vi công nghệ và dùng view_file nạp động đúng skill cần thiết, không cần nạp sẵn từ trước để tiết kiệm token tối đa cho các lượt hội thoại sau.

Các thay đổi

Gom nhóm & Dọn dẹp Kế hoạch

  • Cập nhật workflow /pl: Tự động gom nhóm lưu file kế hoạch vào thư mục DOCS/Planning/.

  • Tự động tăng version file kế hoạch planning.
