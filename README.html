Giới thiệu WSR Core

Introduction to WSR Core

WSR Core là nền tảng quản trị Agent độc lập, xây dựng trên Anti-Waste và Minimal Coding. Mục tiêu của bản phát hành này là giảm thiểu "ảo giác" của AI, đồng thời nén dung lượng ngữ cảnh hệ thống nhằm tăng tốc độ phản hồi và độ chính xác của các mô hình ngôn ngữ lớn (LLM) trong quá trình phát triển mã nguồn.

WSR Core is an independent Agent governance platform built upon Anti-Waste and Minimal Coding. The goal of this release is to minimize AI hallucinations and compress system context volume to maximize speed and precision of Large Language Models (LLMs) during source code generation.

Triết lý "Core Governance": Cắt bỏ các phần diễn giải dài dòng, thay bằng cấu trúc quy tắc dạng điện tín, tối ưu hóa các điểm neo hành vi cốt lõi để AI hoạt động chuẩn xác và tiết kiệm tài nguyên.

"Core Governance" Philosophy: Cuts down long explanations, replacing them with telegraphic rules, optimizing core behavioral anchors so AI works precisely while saving resources.

Các tính năng chính

Key Features

  • Quản lý ngữ cảnh thông minh: Cơ chế tự động dọn dẹp và nén ngữ cảnh cũ tránh tràn bộ nhớ. Smart Context Management: Automatically cleans and compresses old context to avoid memory overflow.
  • Giao thức an toàn (Risk Gate): Đánh giá mức độ rủi ro hệ thống trước khi thay đổi tệp tin. Risk Gate Security Protocol: Evaluates system risk level before modifying files.
  • Hỗ trợ công nghệ bản địa (Native-first): Ưu tiên HTML5/CSS gốc, giảm dependency thừa. Native-first Tech stack: Prioritizes native HTML5/CSS, minimizing redundant dependencies.

Ứng dụng thực tiễn trong AI Vibe Coding

Practical Application in AI Vibe Coding

Vibe Coding đề cập đến phong cách lập trình mà con người chỉ định hướng ý tưởng chung và để AI tự do thực thi mã nguồn. Nếu không có bộ kiểm soát như WSR, AI sẽ nhanh chóng sa vào bẫy quá đà thiết kế (over-engineering), viết mã rác hoặc làm hỏng các hàm dùng chung hiện tại.

Vibe Coding refers to a programming style where humans direct general ideas and let AI freely execute the code. Without WSR controls, AI quickly falls into over-engineering traps, writing junk code or breaking existing shared functions.

Detailed Installation Guide

WSR Core dùng adapter v2, router chứa source root tuyệt đối và marker có kiểm chứng để Codex, Gemini, Antigravity và IDE agent khác nạp nhất quán.

WSR Core uses v2 adapters, an absolute source router, and a verified marker for consistent loading across Codex, Gemini, Antigravity, and other IDE agents.

Các bước cài đặt thủ công

Manual Installation Steps

  1. Chạy source Doctor 32 gate.Run the 32-gate source Doctor.
  2. Chọn adapter và target root; generic bắt buộc workspace root tường minh.Select an adapter and target root; generic requires an explicit workspace root.
  3. Xem dry-run, apply sau phê duyệt, rồi chạy runtime Doctor.Review the dry-run, apply after approval, then run runtime Doctor.

Cài đặt tự động bằng script

Automatic Installation via Script

Ví dụ an toàn cho IDE agent generic trong workspace:

Safe example for a generic IDE agent inside a workspace:

python scripts/wsr_doctor.py --source-mode --strict
python scripts/sync_config.py --adapter generic --target-root <workspace>
python scripts/wsr_doctor.py --runtime-mode --adapter generic --target-root <workspace> --strict

Detailed Usage Instructions

WSR Core tích hợp sẵn các lệnh điều hướng hành vi (Slash Commands) và các gói kỹ năng (Skills) chuyên dụng.

WSR Core integrates built-in behavior-guiding Slash Commands and specialized Skills.

Chi tiết các Lệnh Slash Commands

Slash Commands Details

  • `/br` (Brainstorm): Phân tích và đề xuất giải pháp trực tiếp trong chat. Tuyệt đối KHÔNG tự ý sửa code khi đang thảo luận `/br`. `/br` (Brainstorm): Analyzes and proposes solutions directly in chat. Strictly NO arbitrary code modifications during `/br`.
  • `/pl` (Planning): Lập kế hoạch thiết kế chi tiết và xuất ra tệp tin MD nằm trong `DOCS/Planning/` (không được tự ý viết code khi chưa có sự phê duyệt). `/pl` (Planning): Creates a detailed design plan and writes it to an MD file inside `DOCS/Planning/` (no code writing allowed before approval).
  • `/audit` (Audit chất lượng): Kích hoạt kiểm tra tĩnh chất lượng mã nguồn bằng cách gọi [wsr_audit.py](../.agents/scripts/wsr_audit.py) để phát hiện code rác, mojibake, encoding lỗi. `/audit` (Quality Audit): Triggers static code quality audits by calling [wsr_audit.py](../.agents/scripts/wsr_audit.py) to detect junk code, mojibake, and encoding errors.
  • `/clean` (Dọn dẹp): Chạy [wsr_clean.py](../.agents/scripts/wsr_clean.py) để quét và đề xuất xóa bỏ các file tạm, file backup dư thừa, tối ưu dung lượng đĩa. `/clean` (Clean): Runs [wsr_clean.py](../.agents/scripts/wsr_clean.py) to scan and propose deletion of temporary and redundant backup files.
  • `/wsr` (Đóng gói): Kích hoạt [pack_wsr.py](../.agents/scripts/pack_wsr.py) tự động thu thập cấu hình hiện tại, nâng phiên bản và đóng gói thành tệp `.zip` an toàn. `/wsr` (Package): Activates [pack_wsr.py](../.agents/scripts/pack_wsr.py) to bundle the current config, bump the version, and zip it securely.
  • `/reload` (Nạp lại): Refresh lại toàn bộ cấu hình rules, skills và memory cục bộ mà không làm mất lịch sử hội thoại. `/reload` (Reload): Refreshes the local configuration rules, skills, and memory without losing conversation history.

Các Gói Kỹ Năng Nhúng Sẵn (Embedded Skills)

Embedded Skills

  • wsr-audit: Quét over-engineering, code rác, dependency thừa thãi và đề xuất thay bằng thư viện tiêu chuẩn (stdlib/native). wsr-audit: Scans for over-engineering, junk code, and redundant dependencies, recommending standard libraries (stdlib/native) instead.
  • wsr-debt: Thu thập các comment đánh dấu nợ kỹ thuật `# wsr-debt` hoặc `TODO` vào một ledger tập trung ([debt_ledger.md](../.agents/debt_ledger.md)) để theo dõi. wsr-debt: Gathers technical debt comments marked with `# wsr-debt` or `TODO` into a central ledger ([debt_ledger.md](../.agents/debt_ledger.md)) for tracking.
  • wsr-review: Phân tích Git diff để phát hiện nhanh các thay đổi có nguy cơ phá vỡ logic cũ hoặc thiết kế quá mức cần thiết. wsr-review: Analyzes Git diff to quickly identify changes that risk breaking old logic or representing excessive design.

Mô tả kỹ thuật nâng cao & Logic xử lý

Advanced Technical Description & Processing Logic

WSR Core hoạt động dựa trên sự phối hợp chặt chẽ giữa Manifest (Cấu hình tổng), Checksums (Bảo vệ tính toàn vẹn) và các Script tiện ích.

WSR Core operates on tight coordination between the Manifest (Master Config), Checksums (Integrity Protection), and Utility Scripts.

Chi tiết các thành phần chính

Key Components Detail

  • WSR_MANIFEST.json: Đóng vai trò là nguồn dữ liệu chuẩn (Source of Truth) xác định toàn bộ tệp tin thuộc gói cấu hình WSR. WSR_MANIFEST.json: Acts as the Source of Truth defining all files included in the WSR configuration package.
  • WSR_CHECKSUMS.json: Chứa mã băm SHA-256 của từng file để phục vụ lệnh `wsr_doctor.py` kiểm tra tính toàn vẹn, chống lỗi bảng mã (mojibake) và thay đổi ngoài ý muốn. WSR_CHECKSUMS.json: Contains the SHA-256 hash of each file for `wsr_doctor.py` to verify integrity, protecting against encoding errors (mojibake) and unintended changes.
  • Adapters (adapters/*.json): Cấu hình ánh xạ các biến môi trường và luật tương thích cho từng IDE cụ thể (Cursor, Gemini, Claude...). Adapters (adapters/*.json): Configures environment mappings and compatibility rules for specific IDEs (Cursor, Gemini, Claude...).
