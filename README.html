Hướng dẫn Vận hành WSR 2.4.6 Hardened

⚠️ CẢNH BÁO PHIÊN BẢN:
Tài liệu này thuộc Review Package WSR 2.4.6 nháp. Cấu hình chưa được áp dụng trực tiếp lên hệ thống runtime global của Agent.

1. Thư mục cấu hình chuẩn (Canonical Folder)

Hệ thống WSR sử dụng thư mục .agents/ làm đường dẫn cấu hình và lưu trữ runtime chính thức. Đường dẫn .codex/ chỉ được duy trì làm bí danh (compatibility alias) phục vụ tương thích ngược.

2. Quy trình lập kế hoạch `/pl` (Planning Mode)

Khi thiết kế kế hoạch, Agent tạo và ghi nhận tại: DOCS/Planning/planning_[tên_nhiệm_vụ].md.

Kế hoạch chỉ chứa mã giả (pseudocode), scope tác động, memory anchors, và checklist. Tuyệt đối không chứa mã nguồn đầy đủ để tối ưu hóa context window.

3. Chế độ quét `/audit`

Chế độ Ý nghĩa Lệnh thực thi nháp
quick Quét nhanh các file thay đổi (Git changed) python scripts/wsr_audit.py --changed
deep Quét sâu toàn bộ package review python scripts/wsr_audit.py --mode deep
debt Quét nợ kỹ thuật (TODO/FIXME/wsr-debt) python scripts/wsr_debt_scanner.py --check
drift So sánh drift cấu hình giữa nháp và global python scripts/sync_config.py (Không có --apply)
doctor Kiểm tra tính nhất quán toàn diện hệ thống Chạy quét manifest và tệp tin bắt buộc

4. Script Inventory (Danh mục script)

  • scripts/wsr_debt_scanner.py: Bộ quét nợ kỹ thuật chống false positive.
  • scripts/wsr_audit.py: Bộ kiểm thử tĩnh lỗi mã hóa (mojibake) và cú pháp.
  • scripts/wsr_clean.py: Dọn dẹp an toàn dựa trên whitelist.
  • scripts/pack_wsr.py: Đóng gói package review thành zip.
  • scripts/sync_config.py: Đồng bộ nháp lên global khi có --apply.

5. Nạp Skill JIT (Dynamic Skill Loading)

Hệ thống hỗ trợ nạp cấu hình lazy-load. Nếu skill file chứa các khối phân tách dạng <!-- START_BLOCK: [TÊN] -->, Agent chỉ thực hiện nạp block liên quan trực tiếp đến tác vụ hiện tại để tiết kiệm tối đa token.
